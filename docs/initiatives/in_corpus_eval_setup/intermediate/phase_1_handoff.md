## Key Implementation Details

### Chunk Sampling

- Uses Azure AI Search `search_text="*"` to retrieve all chunks
- Randomly samples chunks for diversity
- Handles empty index and missing index gracefully

### Query Generation

- Uses LLM with temperature=0.7 for diversity
- Prompts LLM to generate In-Corpus queries from chunk content
- Handles individual query failures gracefully (continues with remaining queries)
- Removes quotes from LLM output

### Error Handling

- Retry logic with exponential backoff for Azure AI Search operations
- Graceful handling of empty index, missing index, LLM failures
- Individual query failures don't stop the entire process

## Phase 2 Entry Point

### Implementation Target

**Component**: Dataset Generator  
**File**: `evaluations/_shared/scripts/generate_eval_dataset.py`  
**Input**: `evaluations/in_corpus_eval/inputs/eval_inputs.json`  
**Output**: `evaluations/in_corpus_eval/dataset/eval_dataset.json`

### Key Requirements

1. **Load eval_inputs.json**
   - Load queries from Phase 1 output
   - Validate structure

2. **Generate Retrieval Query**
   - Use input query as retrieval_query (or LLM sanitization if needed)
   - Support query sanitization/changes if required

3. **Retrieve Ground Truth Context**
   - Use retrieval_query to retrieve context
   - Store retrieved chunks with similarity scores
   - This is the ground truth for BEIR metrics

4. **Query Supabase for Prompts**
   - Query system_prompt: `prompt_type` with `name="{type}_system"`
   - Query structured_prompt: `prompt_type="RAG-enabled"` and `live=TRUE`

5. **Generate RAG Output**
   - Use system_prompt and structured_prompt
   - Use retrieval_query for retrieval
   - Generate output using RAG pipeline

6. **Compute BEIR Metrics**
   - Use ground truth chunk IDs from metadata.source_chunk_ids
   - Compute recall_at_k, precision_at_k, ndcg_at_k
   - Use existing `compute_beir_metrics()` function

7. **Initialize Evaluation Fields**
   - Set LLM evaluation fields to null: `llm_correctness`, `llm_hallucination`, `llm_risk_direction`, `llm_risk_magnitude`
   - Set human evaluation fields to null: `human_correctness`, `human_hallucination`, `human_risk_direction`, `human_risk_magnitude`

8. **Save to eval_dataset.json**
   - Save complete evaluation dataset with all required fields

### Output Format

The dataset generator should produce `eval_dataset.json` with the following structure:

```json
[
  {
    "id": "eval_001",
    "content": {
      "input": "string - input for RAG",
      "retrieval_query": "string - sanitized/changed query",
      "context": {
        "retrieved_chunks": [
          {
            "chunk_id": "chunk_1",
            "similarity_score": 0.95,
            "chunk_text": "...",
            "metadata": {...}
          }
        ]
      },
      "system_prompt": "string - system prompt",
      "structured_prompt": "string - structured prompt",
      "output": "string - RAG output",
      "beir_metrics": {
        "recall_at_k": 0.8,
        "precision_at_k": 0.6,
        "ndcg_at_k": 0.75
      },
      "llm_correctness": null,
      "llm_hallucination": null,
      "llm_risk_direction": null,
      "llm_risk_magnitude": null,
      "human_correctness": null,
      "human_hallucination": null,
      "human_risk_direction": null,
      "human_risk_magnitude": null
    }
  }
]
```

### Integration Points

1. **Azure AI Search**
   - Use `retrieve_chunks()` from `backend/rag_eval/services/rag/search.py`
   - Use retrieval_query for ground truth context retrieval

2. **Supabase**
   - Query system prompts: `prompt_type` with `name="{type}_system"`
   - Query structured prompts: `prompt_type="RAG-enabled"` and `live=TRUE`
   - Use `QueryExecutor` from `backend/rag_eval/db/queries.py`

3. **RAG Pipeline**
   - Use `generate_answer()` from `backend/rag_eval/services/rag/generation.py`
   - Use system_prompt and structured_prompt from dataset

4. **BEIR Metrics**
   - Use `compute_beir_metrics()` from `backend/rag_eval/services/evaluator/beir_metrics.py`
   - Use ground truth chunk IDs from metadata.source_chunk_ids
