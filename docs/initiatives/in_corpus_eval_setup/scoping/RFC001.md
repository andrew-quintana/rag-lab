# RFC 001 — In-Corpus Evaluation Dataset Generation System

## Context

This RFC defines the technical architecture and implementation plan for the In-Corpus Evaluation Dataset Generation System, as specified in [PRD001.md](./PRD001.md). The system provides an end-to-end pipeline for generating evaluation datasets from Azure AI Search indexed documents, including query generation, dataset creation, and output generation.

**Status**: The system is currently not implemented. This RFC provides the technical design for implementing the complete dataset generation stack.

## Design Overview

The In-Corpus Evaluation Dataset Generation System consists of modular components that work together to generate complete evaluation datasets. The system is designed for:

- **Automation**: Automated query generation and dataset creation
- **Completeness**: All required fields for comprehensive evaluation
- **Ground Truth**: Ground truth retrieval context for accurate BEIR metrics
- **Integration**: Seamless integration with existing RAG and evaluation systems
- **Flexibility**: Support for different evaluation scenarios via configuration

## Component Architecture

### 1. Query Generator (AI Node)

**File**: `evaluations/{eval_name}/query_generator.py`

**Purpose**: Generate diverse, effective queries from Azure AI Search indexed documents using LLM.

**Input**:
- Azure AI Search index with chunks and embeddings
- Configuration for LLM provider

**Output**: `eval_inputs.json`
```json
[
  {
    "input": "string - query for RAG evaluation",
    "metadata": {
      "source_chunk_ids": ["chunk_1", "chunk_2"],
      "document_id": "doc_123",
      "generation_method": "ai_generated"
    }
  }
]
```

**Key Functions**:
```python
def generate_queries_from_index(
    config: Config,
    num_queries: int = 10,
    sample_chunks: Optional[List[Chunk]] = None
) -> List[Dict[str, Any]]:
    """
    Generate diverse queries from Azure AI Search index.
    
    Args:
        config: Application configuration
        num_queries: Number of queries to generate
        sample_chunks: Optional list of chunks to use as seed (if None, samples from index)
        
    Returns:
        List of query dictionaries with input and metadata
    """
```

**Implementation Details**:
- Sample chunks from Azure AI Search index
- Use LLM to generate diverse queries based on chunk content
- Ensure all queries are "In-Corpus" (answerable from indexed documents)
- Include metadata linking queries to source chunks for ground truth
- Save to `evaluations/{eval_name}/in_corpus/eval_inputs.json`

### 2. Dataset Generator

**File**: `evaluations/{eval_name}/generate_eval_dataset.py`

**Purpose**: Generate complete evaluation dataset from eval_inputs.json.

**Input**: `eval_inputs.json` from query generator

**Output**: `eval_dataset.json`
```json
[
  {
    "id": "eval_001",
    "content": {
      "input": "string - input for RAG",
      "retrieval_query": "string - sanitized/changed query",
      "context": {
        "retrieved_chunks": [
          {
            "chunk_id": "chunk_1",
            "similarity_score": 0.95,
            "chunk_text": "...",
            "metadata": {...}
          }
        ]
      },
      "system_prompt": "string - system prompt",
      "structured_prompt": "string - structured prompt",
      "output": "string - RAG output",
      "beir_metrics": {
        "recall_at_k": 0.8,
        "precision_at_k": 0.6,
        "ndcg_at_k": 0.75
      },
      "llm_correctness": null,
      "llm_hallucination": null,
      "llm_risk_direction": null,
      "llm_risk_magnitude": null,
      "human_correctness": null,
      "human_hallucination": null,
      "human_risk_direction": null,
      "human_risk_magnitude": null
    }
  }
]
```

**Key Functions**:
```python
def generate_eval_dataset(
    eval_inputs_path: str,
    output_path: str,
    config: Config
) -> None:
    """
    Generate evaluation dataset from eval_inputs.json.
    
    Args:
        eval_inputs_path: Path to eval_inputs.json
        output_path: Path to save eval_dataset.json
        config: Application configuration
    """
```

**Implementation Details**:
- Load eval_inputs.json
- For each input:
  - Generate retrieval_query (LLM call if sanitization needed, otherwise same as input)
  - Retrieve context using retrieval_query (ground truth for BEIR)
  - Query Supabase for system_prompt (prompt_type with name="{type}_system")
  - Query Supabase for structured_prompt (prompt_type="RAG-enabled", live=TRUE)
  - Generate output using RAG pipeline
  - Compute BEIR metrics using ground truth chunk IDs from metadata
  - Initialize LLM/human evaluation fields as null
- Save to `evaluations/{eval_name}/in_corpus/eval_dataset.json`

### 3. Output Generator

**File**: `evaluations/{eval_name}/generate_eval_outputs.py`

**Purpose**: Generate RAG outputs for evaluation dataset entries.

**Key Functions**:
```python
def generate_outputs_for_dataset(
    dataset_path: str,
    config: Config
) -> None:
    """
    Generate RAG outputs for evaluation dataset entries.
    
    Args:
        dataset_path: Path to eval_dataset.json
        config: Application configuration
    """
```

**Implementation Details**:
- Load evaluation dataset
- For each entry:
  - Use system_prompt and structured_prompt from dataset
  - Use retrieval_query for retrieval
  - Use retrieved context for generation
  - Generate output using RAG pipeline
  - Update output field in dataset
- Save updated dataset

### 4. System Prompt Management

**Files**:
- Migration: `infra/supabase/migrations/0016_add_system_prompts.sql`
- Script: `evaluations/{eval_name}/create_system_prompts.py` (optional helper)

**Purpose**: Create and manage system prompts in Supabase.

**Naming Convention**:
- For `prompt_type="rag"`, system prompt has `name="rag_system"`
- For `prompt_type="RAG-enabled"`, system prompt has `name="rag_enabled_system"`
- Allows filtering with `name LIKE '%_system'` or `name CONTAINS '_system'`

**Database Schema**:
- System prompts stored in `public.prompts` table
- Same schema as regular prompts (id, version, prompt_type, name, prompt_text, live, created_at)
- Query by prompt_type and name to retrieve system prompts

### 5. Terminology Update: risk_impact → risk_magnitude

**Files to Update**:
- `backend/rag_eval/core/interfaces.py`
- `backend/rag_eval/services/evaluator/meta_eval.py`
- `backend/rag_eval/services/evaluator/risk_impact.py`
- All other files with `risk_impact` references

**Changes**:
- `JudgeEvaluationResult.risk_impact` → `risk_magnitude`
- `MetaEvaluationResult.ground_truth_risk_impact` → `ground_truth_risk_magnitude`
- `JudgePerformanceMetrics.risk_impact` → `risk_magnitude`
- All variable names, function parameters, database columns, etc.

## Database Changes

### System Prompts Table

System prompts use existing `public.prompts` table with naming convention:
- `prompt_type`: Type of prompt (e.g., "rag", "RAG-enabled")
- `name`: System prompt name with `{type}_system` convention
- `prompt_text`: System prompt content
- `live`: Boolean flag for active version

### Migration

Create migration `0016_add_system_prompts.sql` to:
- Insert basic system prompts for RAG-enabled prompts
- Ensure at least one system prompt exists for each prompt type used

## Integration Points

### Azure AI Search
- Query generator samples chunks from index
- Dataset generator uses retrieval for ground truth context

### Supabase
- Query system prompts by prompt_type and name
- Query structured prompts by prompt_type="RAG-enabled" and live=TRUE

### RAG Pipeline
- Use existing `retrieve_chunks()` for retrieval
- Use existing `generate_answer()` for output generation
- Use existing prompt loading functions

### Evaluation System
- Generated datasets compatible with existing evaluation system
- BEIR metrics computed using existing `compute_beir_metrics()` function

## Error Handling

- Handle missing prompts gracefully (raise clear errors)
- Handle empty Azure AI Search index (return empty results)
- Handle LLM generation failures (retry with backoff)
- Handle invalid eval_inputs.json (validate before processing)

## Testing Strategy

- Unit tests for query generator
- Unit tests for dataset generator
- Unit tests for output generator
- Integration tests with mocked Azure AI Search and Supabase
- End-to-end test with sample data

## Dependencies

- Azure AI Search for embeddings and retrieval
- Supabase for prompt storage and querying
- Existing RAG pipeline components
- LLM providers for query generation and sanitization
- Evaluation system for BEIR metrics computation
