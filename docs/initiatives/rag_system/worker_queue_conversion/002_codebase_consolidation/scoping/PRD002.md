# PRD 002 — Codebase Consolidation & Production Readiness

## Problem & Goals

### Problem Statement
The current Azure Functions deployment structure contains duplicate backend code, creating several maintenance and deployment issues:

1. **Code Duplication**: The `infra/azure/azure_functions/backend/src/` directory contains a full copy of the backend code, duplicating the source code in `backend/src/`. This creates two sources of truth and increases maintenance burden.

2. **Build Process Complexity**: The build script (`build.sh`) copies the entire backend codebase during Git deployment, adding unnecessary steps and increasing deployment time.

3. **Risk of Code Drift**: Having duplicate code increases the risk of the copied code diverging from the source, leading to bugs and inconsistencies.

4. **Deployment Package Size**: The deployment package includes a full copy of the backend, making it larger than necessary and potentially impacting cold start times.

5. **Configuration Fragmentation**: Environment variable loading and configuration management is split between `.env.local`, `local.settings.json`, and Azure Function App settings, creating confusion about which configuration takes precedence.

6. **Test Infrastructure Duplication**: Test utilities and fixtures may be duplicated or inconsistently organized between local and cloud testing environments.

### Goals
The Codebase Consolidation initiative must accomplish:

1. **Eliminate Code Duplication & Reorganize Structure**
   - Move Azure Functions from `infra/azure/azure_functions/` to `backend/azure_functions/`
   - Functions import directly from `src` (no path manipulation needed)
   - Single source of truth for all backend code
   - Self-contained functions with simple, maintainable imports

2. **Streamline Deployment Process**
   - Simplify build script to eliminate code copying
   - Reduce deployment package size
   - Improve deployment reliability and speed

3. **Consolidate Configuration Management**
   - Unify environment variable loading strategy
   - Clarify precedence between `.env.local`, `local.settings.json`, and Azure settings
   - Standardize configuration across local and cloud environments

4. **Consolidate Test Infrastructure**
   - Unify local and cloud test execution workflows
   - Consolidate test fixtures and utilities
   - Register pytest markers to eliminate warnings
   - Create comprehensive test execution scripts

5. **Production Readiness Validation**
   - Complete Phase 5 testing in both local and cloud environments
   - Resolve all identified issues from Phase 5 testing
   - Validate performance requirements
   - Ensure production deployment is stable and reliable
